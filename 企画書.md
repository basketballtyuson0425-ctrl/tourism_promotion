# 伊勢観光分析 企画書

## 1. 企画概要

「伊勢観光分析」は、伊勢志摩に関する YouTube 上の旅行・観光コンテンツを分析し、海外旅行者に向けた観光プロモーションの改善に活用するためのプロトタイプである。

本企画では、伊勢志摩の観光資源が海外旅行者にどのように受け取られているかを把握し、今後の情報発信や施策立案につなげることを目的とする。

## 2. 背景

近年、日本全体では訪日外国人観光客の回復が進んでいる。京都、ニセコ、富士山周辺など、SNS や動画で話題化しやすい地域では、海外からの関心が高まりやすい状況がある。

一方で、伊勢志摩には伊勢神宮、英虞湾、海産物などの豊富な観光資源があるにもかかわらず、海外向けの発信や SNS 上での広がりには改善余地があると考えた。

そこで、伊勢志摩を対象地域として選定し、旅行者目線の投稿を分析することで、より効果的な観光プロモーションにつなげる仕組みを検討した。

## 3. 対象地域の選定理由

伊勢志摩には、海外旅行者に訴求しやすい観光資源が複数存在する。

- 伊勢神宮: 日本文化を体験できる代表的な観光資源
- 英虞湾・リアス海岸: 自然景観、クルーズ、リゾート体験につながる観光資源
- 食文化: 伊勢海老、牡蠣、海鮮、松阪牛など、旅行目的になりやすい食の魅力

これらの魅力は十分にあるものの、海外旅行者が実際に何に関心を持ち、どのように発信しているかを継続的に把握する仕組みが必要だと考えた。

## 4. 現状課題

伊勢志摩の観光プロモーションにおける課題は、以下の通りである。

- 自治体や公式サイトからの発信が中心になりやすい
- 単発の PR で終わりやすい
- 海外旅行者が何に反応しているか把握しにくい
- YouTube や SNS 上の投稿傾向を継続的に確認しにくい

特に、海外旅行者の視点で「何が魅力として伝わっているのか」を把握することが重要である。

## 5. 分析対象

本企画では、分析対象を YouTube に絞る。

YouTube を選定した理由は、旅行動画では訪問先、体験内容、食事、景色、移動の流れなどが具体的に表現されやすく、観光地の魅力や課題を読み取りやすいためである。

将来的には Instagram や TikTok なども分析対象に追加できるが、今回のプロトタイプでは、まず説明しやすく実現可能な範囲として YouTube を中心に設計する。

## 6. 比較対象

比較対象はニセコに絞る。

ニセコは、海外からの認知度が高く、自然観光、滞在型観光、海外向け発信の点で参考にしやすい地域である。

| 地域 | 特徴 | 参考にする観点 |
|---|---|---|
| ニセコ | 高級リゾート、自然体験、海外人気 | 海外目線の発信、滞在型観光の見せ方 |
| 伊勢志摩 | 文化、自然、食 | 魅力は多いが、海外向けの見せ方に改善余地がある |

ニセコをそのまま模倣するのではなく、海外旅行者に伝わりやすい発信方法を比較対象として参考にする。

## 7. 解決方針

本プロトタイプでは、次の3点を実現することを目指す。

1. 伊勢志摩に関する YouTube 投稿の傾向を整理する
2. 海外旅行者が反応しやすい観光テーマを把握する
3. 分析結果をもとに、次に取るべき情報発信案を提示する

単に投稿を集めるだけでなく、分析結果を観光プロモーションの改善案につなげる点を重視する。

## 8. システム概要

「伊勢観光分析」では、以下の流れを想定する。

1. 伊勢志摩に関する YouTube 動画や投稿情報を収集する
2. タイトル、説明文、再生数、コメント、使用されている言葉を整理する
3. 人気のある観光テーマや、海外旅行者が反応している内容を確認する
4. 画面上で分かりやすく表示する
5. 観光プロモーションの改善案を提示する

今回の提出では、実データ連携ではなく、サンプルデータを用いて画面と考え方を示す。

## 9. デモで示す内容

デモでは、完成したシステムではなく、課題へのアプローチと画面イメージを示す。

- 分析の全体像が分かるトップ画面
- 人気キーワードを確認できる画面
- 文化、自然、食などの観光テーマ別の傾向を確認できる画面
- ニセコとの比較を確認できる画面
- 分析結果から発信アイデアを提示する画面

これにより、観光担当者が「どの魅力を、どのように発信すべきか」を考えるための補助ツールとして位置づける。

## 10. 工夫した点

本企画で工夫した点は、以下の通りである。

- 対象地域を伊勢志摩に絞り、課題設定を明確にした
- 分析対象を YouTube に絞り、実現可能な範囲にした
- 比較対象をニセコに絞り、説明しやすい構成にした
- 観光資源を「文化・自然・食」に分け、整理しやすくした
- 分析結果を、単なるデータ表示ではなく施策案につなげる構成にした
- 技術の完成度だけでなく、課題の捉え方と改善の流れが伝わるようにした

## 11. 使用技術

提出用デモは、HTML・CSS・JavaScript の最小構成で作成する。

- HTML: 画面構成
- CSS: デザイン、レイアウト
- JavaScript: 表示内容の切り替えやデータ表示

複雑な仕組みを増やしすぎず、面接時に実装内容を説明できる範囲で構成する。

## 12. 今回の対象外

限られた期間で提出可能な形にするため、以下は今回の実装対象外とする。

- YouTube API との本格連携
- 大量データの自動収集
- AI による完全自動分析
- ログイン機能
- 本番運用を前提としたサーバー構築

これらは今後の拡張要素として位置づけ、今回の提出では企画意図と画面イメージを優先する。

## 13. 面接時に伝える要点

面接時には、以下の流れで説明する。

1. 伊勢志摩には観光資源が多い一方で、海外向け発信には改善余地がある
2. YouTube の旅行投稿を分析することで、海外旅行者が何に関心を持っているかを把握できる
3. 比較対象としてニセコを設定し、海外向けに伝わりやすい見せ方を参考にした
4. デモでは、人気キーワード、観光テーマ、比較結果、発信アイデアまでを一連の流れで示す
5. 今回は完成品ではなく、課題の捉え方と解決までの流れを伝えるプロトタイプとして作成した

## 14. 今後の改善案

今後は、以下のような発展を想定している。

- 実際の YouTube データを取り込む
- コメント内容から海外旅行者の感想を分類する
- Instagram や TikTok も分析対象に追加する
- 季節ごとの関心の変化を確認できるようにする
- 観光担当者向けのレポート出力機能を追加する

まずは小さく作成し、分析対象や機能を段階的に広げていく方針である。
