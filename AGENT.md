# AGENT.md

## 絶対遵守: TODO.md の自律管理

このリポジトリでAIエージェントが作業するときは、`TODO.md` の自律管理を最優先ルールとします。

- ユーザーから明示的に指示されなくても、タスクを実行したら必ず `TODO.md` を確認し、必要に応じて更新する
- 作業完了後は、完了したタスクを `TODO.md` 上で完了扱いにし、次にやるべき作業を1〜3個に絞って書く
- 新しい作業方針、保留事項、次回以降の実装候補が出た場合は、チャットだけで終わらせず `TODO.md` に残す
- コンテキストが圧迫されたり、セッションが変わったりしても、`TODO.md` を見れば次に何をすべきか分かる状態を維持する
- `TODO.md` はローカル専用ファイルであり、Gitにコミット・pushしない
- `.gitignore` ではなく `.git/info/exclude` による除外を維持し、GitHubに `TODO.md` の存在や内容を出さない

以後、AIエージェントはすべての作業で「実装・確認・TODO更新」を1セットとして扱います。

このリポジトリでAIエージェントが作業するときの公開用ガイドです。

目的は、プロトタイプを読みやすく保ちながら、企画意図・画面構成・実装方針が伝わる状態にすることです。

## Project Goal

このプロジェクトは、観光プロモーションを支援するための画面プロトタイプです。

SNSや動画などのユーザー投稿コンテンツをもとに、観光地の関心傾向を整理し、情報発信の改善案につなげることを想定しています。

## Working Policy

- まず動く形を優先する
- 画面を見れば目的が分かる構成にする
- 変更理由を説明できる範囲で実装する
- 複雑な仕組みより、読みやすさと再現しやすさを優先する
- 関係のないファイルは変更しない
- 作業ごとに `TODO.md` を更新し、次に取る行動を明確にする

## Tech Stack

このプロジェクトは、HTML・CSS・JavaScript の最小構成を基本にします。

- HTML
- CSS
- JavaScript

Reactなどのフレームワークは利用しません。

既存の `.jsx` ファイルがある場合は、提出・公開しやすい形にするため、必要に応じて通常の HTML・CSS・JavaScript に変換します。

## Implementation Guidelines

- 変数名や関数名で役割が分かるようにする
- コメントは必要な場所だけにする
- 1つの変更で多くの目的を詰め込みすぎない
- UI文言は、見る人がすぐ理解できる短い表現にする
- ダミーデータを使う場合は、実データではなくサンプルであることが分かるようにする
- 実装後は動作確認と `TODO.md` 更新まで行う

## Demo Priorities

提出時や説明の場で見せやすいように、次の順番で整えます。

1. 画面全体の流れが分かること
2. 分析結果の見せ方が分かること
3. 分析から施策提案までのつながりが分かること
4. 今後追加できる機能が説明できること

## Repository Notes

- 公開リポジトリに置く内容は、第三者が読んでも問題ない表現にする
- 個人用メモ、選考対策メモ、企業分析メモはコミットしない
- ローカルだけで使うメモは `.local/` に置く
- `TODO.md` はローカル管理専用とし、GitHubへ公開しない
